    Taller 4: Documentación con MkDocs#

    Bienvenido al taller de documentación técnica usando MkDocs. Este proyecto demuestra cómo crear documentación profesional para proyectos de software libre.

    ¿Qué es MkDocs?#

    MkDocs es una herramienta de código abierto que permite generar sitios web estáticos para documentar proyectos de software. Está orientada a la simplicidad, utiliza archivos Markdown para redactar el contenido y es altamente personalizable mediante temas y plugins.

    Características principales#

    Característica Descripción
    Markdown Escritura en lenguaje de marcado simple y ampliamente adoptado
    Temas Múltiples temas disponibles, incluyendo Material for MkDocs
    Búsqueda Búsqueda de texto en todo el sitio
    Navegación Navegación automática basada en estructura de archivos
    Despliegue Fácil publicación en GitHub Pages, Netlify, etc.

    Comandos útiles#

    Comandos principales

    • mkdocs serve - Inicia el servidor de desarrollo local
    • mkdocs build - Construye el sitio estático
    • mkdocs gh-deploy - Despliega en GitHub Pages
    • mkdocs new [proyecto] - Crea un nuevo proyecto

    Estructura del proyecto#

    Taller4/
├── mkdocs.yml          # Configuración principal
├── docs/               # Documentación en Markdown
│   ├── index.md        # Página principal
│   ├── guias/          # Guías de usuario
│   └── referencia/     # Documentación técnica
└── site/               # Sitio generado (ignorado por Git)
