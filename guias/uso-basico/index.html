    Uso Básico de MkDocs#

    Esta guía te enseñará los conceptos básicos para usar MkDocs y crear documentación efectiva.

    Comandos esenciales#

    Servidor de desarrollo#

    El comando más importante para desarrollo:

    mkdocs serve

    Este comando: - Inicia un servidor local en http://127.0.0.1:8000 - Recarga automáticamente cuando cambias archivos - Muestra errores en tiempo real

    Construir el sitio#

    Para generar el sitio estático:

    mkdocs build

    Esto crea la carpeta site/ con todos los archivos HTML.

    Desplegar en GitHub Pages#

    mkdocs gh-deploy

    Este comando: - Construye el sitio - Lo sube a la rama gh-pages - Lo hace disponible en GitHub Pages

    Convenciones de nombres#

    • Usa kebab-case para nombres de archivos: mi-pagina.md
    • Usa espacios en títulos de navegación
    • Mantén una jerarquía lógica en las carpetas

    Flujo de trabajo recomendado#

    1. Planifica la estructura de tu documentación
    2. Crea los archivos Markdown
    3. Configura la navegación en mkdocs.yml
    4. Prueba localmente con mkdocs serve
    5. Revisa y mejora el contenido
    6. Despliega con mkdocs gh-deploy
